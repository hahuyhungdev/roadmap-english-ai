# IELTS Speaking Topic Module Structure

## Learner Profile

- **Mode:** self-study only, without a teacher
- **Entry level:** IELTS band 5-6 / B1-B2
- **Target level:** IELTS Speaking band 6-7
- **Main audience:** Vietnamese IT professionals or software engineers
- **Main goal:** speak clearly, naturally, and professionally about IELTS topics without sounding overly academic

The course should train learners to speak earlier and retry often. The goal is not to memorize long word lists or essay-style answers. The goal is to build usable speaking language through context, collocations, chunks, patterns, shadowing, and self-review.

---

## Design Principles

1. **Speaking-first**
   Every section must eventually lead to a spoken answer. If a vocabulary item, grammar pattern, or idea cannot be used in a real answer, it should be removed or simplified.

2. **Natural professional English**
   Learners are IT professionals, so the language should sound clear and mature, not theatrical or overly academic. Prefer high-frequency B2 language over rare C1/C2 words.

3. **Context before memorization**
   Vocabulary should appear inside useful situations: work, daily life, technology, meetings, learning, problem solving, and personal experience.

4. **Small input, repeated output**
   Each module should give enough input to speak better, then push the learner to record, review, and retry.

5. **Self-study friendly**
   Every activity must include enough guidance for a learner to complete it alone: examples, answer frames, checklists, and retry tasks.

---

## Module Overview

Each topic module has 6 parts:

```text
Part 1  Topic Activation
Part 2  Vocabulary In Context
Part 3  Core Idea Bank
Part 4  Speaking Patterns, Grammar & Expression Tips
Part 5  Guided Speaking Practice
Part 6  Shadowing, Retry & Self-Review
```

This is the **learner-facing order**. Learners should build usable words, chunks, and sentence material before reading the full idea bank. However, authors should still design the core ideas first internally, then build vocabulary from those ideas.

---

## Part 1 -- Topic Activation

Purpose: help the learner discover what they can already say before learning new material.

Include:

- 3 warm-up questions from IELTS Speaking Part 1
- 1 baseline recording task
- 1 quick self-check: idea problem, vocabulary problem, grammar problem, pronunciation problem, or fluency problem

Example:

```text
Topic: Technology

Warm-up questions:
1. Do you use technology every day?
2. What app or device do you use the most?
3. Has technology changed the way you work or study?

Baseline task:
Record a 60-second answer to: "Do you use technology a lot in your daily life?"

Self-check:
What was hardest: finding ideas, finding words, making sentences, pronunciation, or speaking without long pauses?
```

---

## Part 2 -- Vocabulary In Context

Purpose: build a stronger vocabulary base, but always through context, collocation, chunking, and usable sentence patterns.

This part should be about **1.5x heavier** than a normal speaking lesson, because vocabulary is a major blocker for band 5-6 learners. However, it must not become a dry word list.

Each topic should include:

- **8-10 word families**
- **20-25 core collocations and chunks**
- **8-10 useful topic verbs**
- **8-10 useful adjectives/adverbs**
- **8-10 common Vietnamese learner mistakes**
- **5-8 IT/professional-use phrases** when the topic allows it

### 2a. Word Families

Teach a small number of flexible word families, not random advanced words.

Format:

```text
Word family: automation / automate / automated

Meaning:
automation = tu dong hoa

Natural sentence:
Automation helps companies reduce manual work.

IT/professional sentence:
In software testing, automation helps engineers check the same user flow repeatedly.

Common mistake:
Avoid: make automation for everything
Better: automate repetitive tasks / use automation for repetitive tasks
```

### 2b. Collocations & Chunks

Teach phrases as ready-to-use speaking units.

Format:

```text
Phrase:
rely heavily on digital tools

Meaning:
phu thuoc kha nhieu vao cong cu so

Pattern:
rely heavily on + noun

Natural sentence:
I rely heavily on digital tools for work and communication.

IT/professional sentence:
As a developer, I rely heavily on GitHub, Slack, and documentation sites.

Common mistake:
Avoid: depend technology
Better: depend on technology / rely on technology
```

Preferred phrase types:

- verb + noun: solve a problem, manage tasks, reduce manual work
- adjective + noun: practical tool, reliable connection, repetitive task
- adverb + adjective: highly useful, deeply connected, increasingly common
- fixed chunks: one thing I like about it is..., the main downside is...

### 2c. Useful Topic Verbs

Use verbs that help learners explain actions clearly.

Example for Technology:

```text
use / rely on / improve / reduce / automate / manage / connect / communicate / solve / keep up with
```

Each verb should include one natural sentence and one IT/professional sentence.

### 2d. Useful Adjectives & Adverbs

Prefer natural spoken adjectives over overly academic ones.

Good for Speaking:

```text
useful / practical / reliable / convenient / time-saving / distracting / stressful / efficient / secure / flexible
```

Use carefully:

```text
detrimental / indispensable / unprecedented / pivotal
```

These can appear as recognition vocabulary, but they should not dominate model answers for band 6-7 speaking.

### 2e. Common Vietnamese Learner Mistakes

Every topic should include mistake correction.

Format:

```text
Mistake:
I very depend on my phone.

Better:
I depend on my phone a lot.
I rely heavily on my phone.

Why:
"depend" needs "on"; "very" does not modify verbs.
```

## Part 3 -- Core Idea Bank

Purpose: give the learner enough ideas to answer IELTS Speaking Part 1, Part 2, and Part 3 without memorizing essays.

Learners should read this after the vocabulary section so the ideas feel easier to speak. Each idea should reuse chunks from Part 2.

Each topic should include **6-8 core ideas**. Each idea should have:

- a simple version
- a better spoken version
- an IT/professional example when relevant
- one useful phrase or collocation connected to the idea

Recommended format:

```text
Idea: Technology saves time at work.

Simple version:
It helps people do tasks faster.

Better spoken version:
It helps people automate repetitive tasks and focus on more important work.

IT/professional example:
As a developer, I use automation tools to test the same flow many times, so I can focus more on debugging and product logic.

Useful language:
automate repetitive tasks / focus on higher-value work / save a lot of time
```

Idea categories:

- personal daily life
- work or study
- advantages
- disadvantages
- changes over time
- future prediction
- IT/professional example
- balanced opinion for IELTS Speaking Part 3

---

## Part 4 -- Speaking Patterns, Grammar & Expression Tips

Purpose: help learners turn vocabulary into real answers quickly while reviewing the grammar structures that naturally fit the topic.

This part bridges input and output. It should teach sentence patterns, topic grammar, answer patterns, expression upgrades, and repair strategies.

### 4a. Sentence Patterns

Include high-frequency spoken patterns:

```text
I tend to + verb...
I rely on + noun + to + verb...
One thing I like about X is that...
The main downside is that...
Compared to the past, ...
It has become much easier to...
I would not say X is perfect, but...
The reason I say that is...
What I mean is...
```

Each pattern should include:

- meaning or function
- 2 topic examples
- 1 learner transformation task

Example:

```text
Pattern:
One thing I like about X is that...

Function:
Explain a benefit naturally.

Topic example:
One thing I like about online learning is that I can study at my own pace.

IT example:
One thing I like about AI coding tools is that they help me draft simple code faster.

Task:
Use this pattern to answer: "What technology do you use the most?"
```

### 4b. Topic Grammar In Use

Grammar should be taught as a speaking tool, not as a separate textbook lesson. Each topic should include **3-5 grammar structures** that help learners answer that topic better.

For each grammar point, include:

- function in this topic
- simple formula
- 2 natural speaking examples
- 1 IT/professional example when relevant
- 1 weak sentence -> better sentence upgrade
- 1 short speaking drill

Recommended grammar types for band 5-6 -> 6-7:

```text
Tendency:
People tend to + verb...
People are likely to + verb...

Comparison:
Compared to the past, ...
X is more convenient than Y because...

Cause and effect:
Because of X, ...
This has made it easier to...
This can lead to...

Contrast:
Although X is useful, ...
Having said that, ...

Relative clause:
I use an app that helps me...
It is a tool that allows me to...

Conditional type 2:
If I did not have X, I would...
If companies invested more in X, they could...

Past experience:
I started using it when...
It helped me when I had to...
```

Example:

```text
Grammar:
Tendency

Function:
Talk about common habits or social trends without sounding too absolute.

Formula:
People tend to + verb...

Natural examples:
People tend to check their phones many times a day.
Younger people tend to rely more on digital tools.

IT/professional example:
Developers tend to use documentation and online communities when they get stuck.

Upgrade:
Basic: Young people use phones too much.
Better: Young people tend to spend too much time on their phones.

Speaking drill:
Use "tend to" to answer: "Do people rely on technology too much?"
```

Grammar should be selected by topic. Do not force every grammar point into every topic.

Example topic mapping:

```text
Technology:
tendency / cause-effect / comparison / conditional type 2 / relative clause

Work:
past experience / responsibility language / contrast / cause-effect / modal verbs

Health:
cause-effect / advice with should / tendency / contrast / conditionals

Education:
comparison / cause-effect / relative clause / future prediction / modal verbs
```

### 4c. Answer Patterns

Give repeatable frameworks for IELTS Speaking.

```text
Part 1:
Direct answer -> detail -> small example

Part 2:
What it is -> when/how often -> what you do with it -> personal story -> final feeling

Part 3:
General opinion -> reason -> example -> contrast or condition -> conclusion
```

### 4d. Expression Tips

Teach learners how to sound more natural.

Examples:

```text
Instead of:
Technology is very useful.

Say:
It saves me a lot of time.
It is really practical for my daily work.
It helps me handle repetitive tasks faster.
```

```text
Instead of:
This is detrimental to individuals.

Say:
This can affect people's mental health.
This can make people feel more stressed or distracted.
```

Principle:
Use clear B2-level language first. Add stronger words only when they still sound natural in speech.

### 4e. Upgrade Patterns

Show how to upgrade weak answers step by step.

```text
Basic:
Technology is useful.

Upgrade 1 -- add a reason:
Technology is useful because it saves time.

Upgrade 2 -- add a collocation:
Technology is useful because it helps people automate repetitive tasks.

Upgrade 3 -- add IT/professional context:
As a developer, I use automation tools to handle repetitive tasks, so I can focus on solving more complex problems.
```

Grammar can be part of the upgrade:

```text
Basic:
I use my laptop every day. It is useful.

Upgrade with relative clause:
I use my laptop every day because it is a tool that helps me work, study, and communicate with my team.

Upgrade with contrast:
I use my laptop every day because it is really practical for work. Having said that, I try to take breaks so I do not spend the whole day in front of a screen.
```

### 4f. Repair Strategies

Self-study learners need tools for moments when they get stuck.

Teach phrases like:

```text
What I mean is...
Let me put it another way.
I cannot think of the exact word, but...
A simple example would be...
It is similar to...
```

These help fluency more than memorizing rare vocabulary.

## Part 5 -- Guided Speaking Practice

Purpose: convert ideas, vocabulary, and patterns into IELTS-style answers.

Include:

### 5a. Part 1 -- Short Answers

Use 5-6 realistic Part 1 questions.

Each question should include:

- band 5.5 answer
- upgrade notes
- band 6.5-7 answer
- 1 retry task

Format:

```text
Question:
Do you use technology a lot?

Band 5.5 answer:
Yes, I use technology every day. I use my phone and laptop. They are useful.

Upgrade notes:
- Add "rely on"
- Add IT/work context
- Add one contrast

Band 6.5-7 answer:
Yes, definitely. I rely on technology almost every day, especially my laptop and phone. As a developer, I use digital tools to write code, manage tasks, and communicate with my team. Having said that, I try not to spend too much time scrolling after work.

Retry task:
Answer again using "rely on" and one personal example.
```

### 5b. Part 2 -- Cue Card

Use 1 practical cue card per topic.

Include:

- 1-minute planning bullets
- 2-minute model answer
- stripped-down answer frame
- phrase bank from the model answer
- shadowing version split into short lines

### 5c. Part 3 -- Discussion

Use 4-5 Part 3 questions.

Each question should include:

- answer framework
- model answer
- language breakdown
- simpler version for band 6
- retry prompt

Focus on clarity and development, not academic complexity.

---

## Part 6 -- Shadowing, Retry & Self-Review

Purpose: make self-study measurable and active.

Recommended practice loop:

```text
1. Listen to the model answer.
2. Shadow sentence by sentence.
3. Record your own answer.
4. Review using a short checklist.
5. Pick one improvement.
6. Record the answer again.
```

Self-review checklist:

```text
Content:
- Did I answer the question directly?
- Did I give at least one concrete example?
- Did I develop the answer enough?

Vocabulary:
- Did I use at least 2 topic collocations or chunks?
- Did I avoid repeating the same basic word too much?

Fluency:
- Did I speak without long pauses?
- Did I use a repair phrase when stuck?

Grammar:
- Did I use at least one useful pattern from this topic?
- Did I avoid one common Vietnamese learner mistake?

Pronunciation:
- Which 1-2 words should I pronounce more clearly next time?
```

Retry task format:

```text
Retry goal:
Say the same answer again, but improve only one thing:
- add one example
- use two collocations
- reduce long pauses
- fix one grammar mistake
- pronounce one phrase more clearly
```

## Recommended Authoring Order

Do not write a topic module from Part 1 to Part 6 in order. The learner-facing order shows vocabulary before ideas, but the authoring order should still start from ideas. Use this order for consistency:

1. Choose the topic and IELTS question types.
2. Write the Core Idea Bank first.
3. Build Vocabulary In Context from those ideas.
4. Choose the Topic Grammar In Use from the ideas and model answers.
5. Create Speaking Patterns & Expression Tips from the vocabulary and grammar.
6. Write Guided Speaking Practice answers.
7. Split model answers into shadowing lines.
8. Add self-review and retry tasks.
9. Reorder the final module so learners see Vocabulary In Context before Core Idea Bank.

This keeps the module connected: ideas produce vocabulary, vocabulary produces patterns, patterns produce answers, and answers produce speaking practice.

---

## Quality Bar

A topic module is ready only if a self-study learner can:

- understand the main ideas without teacher explanation
- learn vocabulary through phrases and examples, not isolated lists
- review grammar through topic examples and speaking drills
- answer Part 1, Part 2, and Part 3 questions about the topic
- record and retry their answer with a clear improvement goal
- sound natural, clear, and professional rather than overly academic

Final standard:

```text
Clear enough for IELTS.
Natural enough for real conversation.
Professional enough for an IT learner.
Simple enough for self-study.
```
