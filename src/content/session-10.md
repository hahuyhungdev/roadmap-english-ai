# BUỔI 10: Performance / Cost Scenario

**Level:** B1+ / B2  
**Focus:** Explain a performance-related scenario, discuss optimization choices, analyze cost implications, and link technical decisions to business outcomes using professional B2 language.

## 1) Vocabulary (Expanded & Sophisticated for B2)

- **optimization** (n) – the process of making something work better
  - Performance optimization became our highest priority this quarter.

- **latency** (n) – delay between request and response
  - High latency on mobile networks was affecting user experience.

- **resource consumption** (n) – the amount of memory, CPU, or bandwidth used
  - Heavy images led to excessive resource consumption on user devices.

- **cost efficiency** (n) – achieving good results without wasting money
  - We focused on cost efficiency by reducing unnecessary API calls.

- **scalability cost** (n) – expenses that increase as the system grows
  - Without proper optimization, scalability costs can become very high.

- **budget constraint** (n) – limitation on available money
  - We had to work within tight budget constraints for this feature.

- **return on investment (ROI)** (n) – the benefit gained compared to the cost
  - The optimization delivered a strong ROI by reducing cloud expenses.

- **bottleneck** (n) – a point causing slowdown
  - The image loading process was the main bottleneck.

- **caching strategy** (n) – a plan for storing data to improve speed
  - Implementing an intelligent caching strategy reduced server load significantly.

- **overhead** (n) – extra time, memory, or cost required
  - Complex animations created unnecessary computational overhead.

## 3) Typical Dialogues

### Dialogue 1 – Full Performance/Cost Scenario (3–4 minutes)

**Teacher:** Tell me about a performance issue and how cost played a role in your solution.

**You:**  
Last quarter we had a scenario where the product detail pages were loading slowly on mobile devices, especially in regions with slower networks. The main bottleneck was large uncompressed images and too many simultaneous API requests.

We had two main options: move more processing to the backend (which would increase server costs) or optimize heavily on the frontend. After careful evaluation, we chose the frontend optimization route. We implemented modern image formats (WebP), lazy loading, and a smart caching strategy. We also reduced the number of API calls by combining endpoints.

This approach allowed us to bring down average load time from 5.8 seconds to 1.9 seconds without increasing infrastructure costs. In fact, we actually reduced monthly cloud expenses by 22% because fewer requests reached the servers. The trade-off was spending more development time upfront, but the ROI was excellent — better user experience, higher conversion rates, and lower operational costs.

### Dialogue 2 – Discussing Business Impact

**Teacher:** How did this affect the business?

**You:**  
The faster pages directly improved user satisfaction and reduced bounce rates. From a cost perspective, lowering server load meant we could handle more traffic without upgrading our AWS resources immediately. This gave the company better cost efficiency and more budget flexibility for other features.

## 4) Reading Text

A common performance/cost scenario in enterprise projects occurs when applications grow and page load times start to suffer. In one case, heavy media content and frequent API calls caused high latency and increased cloud costs.

The team decided to focus on frontend optimizations rather than scaling up backend resources. By adopting modern image optimization, implementing aggressive caching, and reducing request overhead, they achieved substantial improvements in speed while actually lowering monthly infrastructure expenses. This solution demonstrated that thoughtful technical decisions can simultaneously enhance user experience and deliver strong cost efficiency — a win-win situation that positively impacted both customer satisfaction and the company’s bottom line.
